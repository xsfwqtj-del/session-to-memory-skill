# 关键词规范

## 提取规则

每条记忆自动打 3-8 个关键词。提取来源：分类名 + 正文标题 + 事件摘要中的核心名词。

## 要求

- 格式: YAML 列表 `[词1, 词2, 词3]`
- 数量: 3-8 个
- 语言: 中文优先，技术名词保留原文
- 粒度: "搜索这个词能精准定位到这条记忆"——不能太宽（"AI"）也不能太窄（整句话）
- **优先使用低热度专有名词**：具体文件名（`settings.json`）、组件名（`Harness`）、错误信号（`SessionStart`）、特定配置项优于通用技术名词（`DeepSeek`/`缓存`/`API`）。示例：
  - ✅ 好: `[Harness, hook, settings.json, SessionStart]` —— 每个词都有具体指向
  - ❌ 差: `[DeepSeek, 缓存, AI, 优化]` —— 太通用，跨主题误关联风险高

## 去口语映射

| 正文中的口语 | 关键词 |
|-------------|--------|
| "那个 hook 没配上" | `hook, 配置缺失` |
| "试了一下发现不行" | `[具体方案名], 排除` |
| "用户说以后不要这样" | `偏好, [具体行为]` |
| "查了一下官方文档" | `[文档名], 外部参考` |

## 去重融合规则

### 会话内去重

```
同一 sessionId + 同一分类 + 标题语义相似（LLM 判断）→ 合并为同一条，补充正文细节
同一 sessionId + 同一分类 + 标题相关但不同事件 → 各自保留，双向写 关联: [[...]]
```

### 跨会话级联判断

第一步由 LLM 判断是否为同一事件。关键词辅助仅用于 LLM 不确定时的兜底——且必须排除高频通用词干扰：

```
第一步 — LLM 判断是否为同一事件：是 / 否 / 不确定
  是 → 合并，更新时间线（追加"## YYYY-MM-DD 更新"段）
  否 → 独立新建
  不确定 → 进入第二步
第二步 — 关键词辅助裁决（Jaccard + 低热度约束）：
  1. Jaccard 相似度 = |交集| / |并集|
  2. 如果 Jaccard ≥ 0.4 且 共同关键词中包含 ≥ 1 个低热度专有名词
     （具体文件名/组件名/错误码/配置项，非"DeepSeek""缓存""API"等通用词）
     → 写入 关联: [[...]]，各自保留
  3. 否则 → 独立新建
```

同一事件的标准：同一触发原因 + 同一结论/结果。仅靠通用技术词重叠不构成关联证据。

## 示例

一条关于配置审计的记忆：
```yaml
关键词: [Harness, hook, settings.json, SessionStart, 配置审计, PreToolUse]
```

一条关于缓存机制的记忆：
```yaml
关键词: [KV Cache, 前缀缓存, token优化, DeepSeek, 命中率]
```
