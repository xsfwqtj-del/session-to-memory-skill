# 记忆文件格式规范

## 输出根目录

当前项目根目录下的 `Mio-SBAgnet—memory/`。首次运行自动创建。

## 文件路径

```
{项目根目录}/Mio-SBAgnet—memory/{分类}/{序号}_{简短描述}.md
```

序号为两位数字 `01` `02`...，按该分类下已有文件数递增。

## Frontmatter 字段

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| 分类 | 是 | 枚举 | 11 类之一 |
| 项目 | 是 | 枚举 | 从 cwd 自动判定，无法判定时填"通用" |
| 时间 | 是 | YYYY-MM-DD HH:MM | 从 JSONL timestamp 提取，精确到分 |
| 主题 | 是 | topic-{sid}-NNN | 主题唯一 ID，步骤 ❸ 分配 |
| 前置主题 | 否 | YAML 列表 | 因果链中的前置主题 ID 列表，单值用单元素列表 `[topic-xxx-NNN]`，无前置用 `[]` |
| 门禁记录 | 是 | YAML 映射 | 11 类逐一标注 ✓（有产出）或 ✗（跳过 + 理由），示例见下方 |
| 关键词 | 是 | YAML 列表 | 3-8 个，中文优先，技术名词保留原文 |
| 关联 | 否 | YAML 列表 | `[[分类/序号_标题]]` 格式的显式引用 |

## 命名规则

```
{分类}/
  01_{简短描述}.md
  02_{简短描述}.md
  ...
```

- 简短描述：5-15 个中文字，能一眼看出内容
- **标题必须自己提炼**：从对话内容中总结核心信息，写出一个完整判断句。禁止直接用用户原话做标题。
- 避免："关于XX的问题""一些想法"、用户的第一句话、含 `---` 或 `→` 的原始粘贴
- 好例子："Harness层hook全部缺失""DeepSeek前缀缓存机制""舍弃中转站改用JSONL回炉"
- 坏例子："搞定。汇总： --- 根因修复 旧: origin →""显示器又黑了""2025 你逗我呢"
