# 转化管线总览

## 6 步管线

```
❶ 预检(脚本) → ❷ 源适配+去噪(脚本) → ❸ 识别主题(LLM)
    → ❹ 聚拢+提取+分类+打标+生成.md(LLM) → ❺ 去重融合(LLM) → ❻ 写入+manifest+aggregate(脚本)
```

## 每步详述

| 步骤 | 执行者 | 输入 | 操作 | 输出 | 参考文件 |
|------|--------|------|------|------|---------|
| ❶ 预检 | 脚本 | JSONL 目录 | 文件检+源识别+概要+处理决策 | 会话清单 | — |
| ❷ 源适配+去噪 | 脚本 | JSONL 原文 | 字段映射→统一格式+去噪+统计工具调用 | 清洁对话流+operationsSummary | `06-extraction-format.md` |
| ❸ 识别主题 | LLM | 清洁对话流 | 遍览全对话→识别独立主题→标注独有关键词 | 主题列表 | `04-topic-identification.md` |
| ❹ 聚拢提取生成 | LLM | 每个主题+清洁对话流 | 关键词回搜聚拢→提取(起因/过程/因果)+分类+打标→生成.md(含操作环境) | .md 文件 | `01-classification.md` `02-file-format.md` `03-keywords.md` |
| ❺ 去重融合 | LLM | 全部已生成 .md + 已有记忆 | 同会话去重+跨会话级联判断→合并/关联/新建+统计提示 | 最终写入清单 | `03-keywords.md` |
| ❻ 写入+汇聚 | 脚本 | 最终写入清单 | Write .md → 从操作环境段汇聚_aggregate → 更新manifest | 记忆文件+摘要+_aggregate | `02-file-format.md` |

## 每步的"该跳过就跳过"

- ❶ 无新会话 → **退出**
- ❷ 对话纯闲聊（<10 exchange，无工具调用）→ **跳过该会话**
- ❸ 无可识别主题 → **标记为空会话，跳过**
- ❹ 主题内容无法提取具体信息 → **跳过该主题**
- ❺ 完全匹配已有记忆 → **合并而不新建**

## 大会话分片策略

步骤 ❶ 标记 `needsSharding: true`（>5MB）时生效：

- **基础 shard 大小**：80 轮 exchange
- **overlap**：前后各 15 轮（shard-1=1-80, shard-2=65-145, shard-3=130-210, ...）
- 步骤 ❸ 对每个 shard 独立识别主题
- **跨 shard 合并**：overlap 区间内识别到的主题与相邻 shard 的主题做关键词对比，重叠 ≥ 50% → 合并为同一主题。合并规则：
  - 保留先出现的 topicId（shard-1 的主题 ID）
  - exchangeRange 取两 shard 的并集（最小值..最大值）
  - uniqueKeywords 取两主题的并集
  - 门禁记录取两主题的 OR（任一 ✓ 则 ✓，均为 ✗ 则保留各自理由中较具体者）
  - primaryWriteOps / sharedWriteOps 取并集去重
- 合并后的主题列表进入步骤 ❸.5

## 错误处理

| 错误 | 处理 |
|------|------|
| JSONL 解析失败 | 跳过该行，记录行号，继续 |
| 关键词不足 3 个 | 从正文强制多提取 |
| 写入目录不存在 | 创建目录 |
| 序号冲突 | 读已有文件数，自动递增 |
| manifest 写入冲突 | .lock 文件排他锁，等待 5 秒后跳过（下次运行补写） |
